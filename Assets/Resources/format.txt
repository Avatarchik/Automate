Описание формата файла description.xml
======================================

Тег: game
Родительский тег: нет
Описание: Содержит информацию об игре, требуемую на момент загрузки меню игр.

Аттрибуты
---------
Название: id
Описание: Внутренний идентификатор игры
Значение по умолчанию: нет

Название: name
Описание: Название игры
Значение по умолчанию: нет

Название: version
Описание: Версия архива игры
Значение по умолчанию: 0

Название: icon
Описание: Путь к картинке игры для меню игр
Базовая директория: корень архива
Значение по умолчанию: нет

Название: iconWidth и iconHeight
Описание: Размеры картинки для меню
Значение по умолчанию: (616, 428)

Название: nameImage
Описание: Путь к картинке с названием игры для меню игр
Базовая директория: корень архива
Значение по умолчанию: нет

Название: width и height
Описание: Размеры картинки с названием
Значение по умолчанию: (0, 0)

Пример
------
<game name="Fruit Coctail" icon="fruit-png" id="fruit" version="6" nameImage="name-png" width="327" height="72">
- внутренний идентификатор игры - fruit;
- название игры - Fruit Coctail;
- иконка находится в файле fruit-png с размерами по умолчанию (616, 428);
- версия архива 6;
- картинка с названием игры для меню находится в файле name-png с размерами (327, 72).

Тег: loadingAnimation
Родительский тег: loading
Описание: Содержит информацию об анимациях, расположенных на экране во время загрузки игры. Считывается вместе с тегом loading.

Атрибуты:
---------
Название: file
Описание: Путь к файлу анимации
Базовая директория: loading
Значение по умолчанию: нет

Название: columns
Описание: Количество столбцов кадров анимации в файле
Значение по умолчанию: 1

Название: rows
Описание: Количество рядов кадров анимации в файле
Значение по умолчанию: 1

ЗАМЕЧАНИЕ: Ширина и высота одного кадра анимации вычисляется путем деления размеров файла с анимацией на количество столбцов и рядов соответственно.

Название: x и y
Описание: Координата верхнего левого угла расположения анимации на экране.
Значение по умолчанию: (0, 0)

Название: width и height
Описание: Размеры анимации на экране.
Значение по умолчанию: (0, 0)

Название: delay
Описание: Скорость анимации (задержка между кадрами в мс)
Значение по умолчанию: 100

Пример
------
<loading background="background-jpg" width="800" height="480" lineX="210" lineY="310">
    <loadingAnimation file="anim-png" columns="6" rows="4" x="320" y="90" width="160" height="230" delay="50"/>
</loading>
- фон окна загрузки находится в файле loading/background-jpg с размерами на экране 800 на 480;
- прогресс бар расположен по координатам (210, 310);
- на экране во время загрузки присутствует всего одна анимация из файла loading/anim-png (в файле 6 столбцов и 4 ряда кадров) с размерами (160, 230), расположенная в точке (320, 90);
- частотам смены кадров - 1 кадр в 50 мс.

Тег: spin
Родительский тег: field
Описание: Содержит настройки вращения барабанов

Атрибуты:
---------
Название: blur
Описание: Признак наличия размытия барабанов при вращении.
Значение по умолчанию: false

Название: speed
Описание: Скорость вращения барабанов (в игровых элементах в промежуток времени, указанный в timerRate). Например, если указано значение 0.5, это значит, что при очередном срабатывании таймера барабан сместится на расстояние, равное 0.5 от высоты элемента.
Значение по умолчанию: 0.5

Название: timerRate
Описание: Частота срабатывания таймера анимации вращения барабана (в секундах)
Значение по умолчанию: 0.05

Название: knockback
Описание: Признак наличия отката при остановке вращения барабана. Если этот признак установлен в true, то перед тем, как барабан остановится, он проедет на один элемент дальше нужного, после чего произойдет откат до требуемого элемента.
Значение по умолчанию: false

Название: delay
Описание: Задержка (в мс) между остановками соседних барабанов. Если это значение отрицательное, то задержка отсчитывается от момента начала торможения предыдущего барабана. Если значение положительное, то задержка отсчитывается от момента полной остановки предыдущего барабана. Если это значение 0, то барабаны останавливаются все вместе.
Значение по умолчанию: 0

Пример
------
<spin blur="false" speed="0.5" timerRate="0.05" knockback="true" delay="-500"/>
- вращение происходит без размытия;
- скорость вращения - полэлемента за 0.05 секунды;
- остановка вращения происходит с откатом;
- остановка каждого следующего барабана происходит через 500 мс после начала остановки предыдущего.

Тег: rollsarea
Родительский тег: field
Описание: Содержит геометрические параметры барабанов

Атрибуты:
---------
Название: x и y
Описание: Координата левого верхнего угла первого барабана
Значение по умолчанию: (0, 0)

Название: xMargin
Описание: Расстояние между барабанами
Значение по умолчанию: 0

Название: yMargin
Описание: Расстояние между элементами барабана
Значение по умолчанию: 0

Название: elementWidth и elementHeight
Описание: Видимые размеры элемента
Значение по умолчанию: (0, 0)

Название: rollsCount
Описание: Количество барабанов на поле
Значение по умолчанию: 5

Название: elementsPerRoll
Описание: Количество видимых элементов на барабане
Значение по умолчанию: 3

Пример
------
<rollsarea x="48" y="33" xMargin="16" yMargin="16" elementWidth="96" elementHeight="96" rollsCount="5" elementsPerRoll="3"/>
- левый верхний угол первого барабана находится в точке (48, 33);
- расстояние между барабанами 16;
- расстояние между элементами в барабане 16;
- видимый размер элемента (96, 96);
- количество барабанов 5, на каждом 3 видимых элемента.

Тег: elements
Родительский тег: field
Описание: Группирует информацию об элементах, используемых в игре.

Атрибуты:
---------
Название: format, count и startIndex
Базовая директория: elements
Описание: Описывает группу файлов, формат имени которых представлен в атрибуте format, который обязательно должен содержать подстановку %d, которая будет заменяться числами.
          Количество файлов определяет атрибут count. Первое число, используемое при считывании файлов, определяется атрибутом startIndex.
          Например, format = %d-png, count = 9, startIndex = 1. Это означает, что должно быть 9 файлов: 1-png, 2-png, ..., 9-png, которые станут элементами с индексами 0, 1, ..., 8 соответственно.
Значение по умолчанию:
startIndex: 0
count: 0
format: нет

Название: width и height
Описание: Размеры считываемых ресурсов
Значение по умолчанию: (0, 0)

Название: x и y
Описание: Координата верхнего левого угла текстуры по умолчанию для всех элементов
Значение по умолчанию: (0, 0)

Пример
------
<elements count="9" format="%d-jpg" startIndex="1" width="96" height="96">
- в игре бывает 9 элементов, находящихся в файлах elements/1-jpg - elements/9-jpg;
- размер картинок (96, 96).

==========================================================================================

Тег: element
Родительский тег: elements
Описание: Предназначен для переопределения некоторых значений для элементов, заданных по умолчанию тегом elements. Так же определяет наличие элемента с определенным индексом.

Атрибуты:
---------
Название: index
Описание: Индекс элемента, который нужно создать. На каждый элемент, который должен присутствовать на поле, должен присутствовать один тег с индексом данного элемента. Индексы начинаются с 0.
          Не может быть больше или равно значения атрибута count тега elements.
Значение по умолчанию: -1

Название: width и height
Описание: Размеры картинки для данного элемента.
Значение по умолчанию: значение атрибутов width и height тега elements.

Название: allowedRolls
Описание: Порядковые номера барабанов, где может отрисовываться элемент при вращении (пример слот secret elixir).
Допустимые значения: [0..n], где n - порядковый номер последнего барабана
Значение по умолчанию: -1 (элемент может отрисовываться при вращении во всех барабанах)

Название: canRepeat
Описание: Позволяет повторяться элементам на барабане вов ремя вращения
Допустимые значения: true,false
Значение по умолчанию: false


Пример:
<element index="3"/>
- создать элемент с индексом 3 (при использовании format = %d-jpg и startIndex = 1 этот элемент будет представлен картинкой elements/4-jpg)
- размеры ресурса используют общие значения из тега elements.

<element index="3" allowedRolls="0,2,4" canRepeat="true"/>
- создать элемент с индексом 3 (при использовании format = %d-jpg и startIndex = 1 этот элемент будет представлен картинкой elements/4-jpg)
- размеры ресурса используют общие значения из тега elements.
- при спине, элемент будет виден только в 1,3 и 5 барабане
- элемент может повторяться при вращении барабана

Тег: animation
Родительский тег: animations
Описание: Содержит данный одной анимации для одного из элементов.

Атрибуты:
---------
Название: index
Описание: Индекс элемента, который нужно создать. На каждый элемент, который должен присутствовать на поле, должен присутствовать один тег с индексом данного элемента. Индексы начинаются с 0.
          Не может быть больше или равно значения атрибута count тега elements.
Значение по умолчанию: -1. 

Название: name
Описание: Название анимации (все анимации считываются отдельно и описаны в своем файле).
Значение по умолчанию: нет

Название: event
Описание: События, при котором данная анимация должна произойти на месте элемента.
Значение по умолчанию: нет

Пример
------
<animations>
    <animation index="8" name="animate_door" event="win"/>
</animations>
- в игре присутствует одна анимация с именем animate_door для элемента с индексом 8, которая происходит при событии win.

Тег: lines
Родительский тег: field
Описание: Группирует информацию о линиях

Атрибуты:
---------
Название: format, count и startIndex
Базовая директория: elements/lines
Описание: Описывает группу файлов, формат имени которых представлен в атрибуте format, который обязательно должен содержать подстановку %d, которая будет заменяться числами.
          Количество файлов определяет атрибут count. Первое число, используемое при считывании файлов, определяется атрибутом startIndex.
          Например, format = line%d-jpg, count = 9, startIndex = 1. Это означает, что должно быть 9 файлов: line1-jpg, line2-jpg, ..., line9-jpg, которые станут линиями с индексами 0, 1, ..., 8 соответственно.
startIndex: 0
count: 9
format: нет

Название: width и height
Описание: Размеры по умолчанию считываемых ресурсов линий
Значение по умолчанию: (0, 0)

Название: x и y
Описание: Координата верхнего левого угла текстуры по умолчанию для всех линий
Значение по умолчанию: (0, 0)

Название: step
Описание: Количество линий, на которое изменяется количество активных линий при нажатии на кнопку Lines.
Значение по умолчанию: 2

Название: showTogether
Описание: Признак того, что выигрышные линии нужно показывать все вместе, а не по очереди.
Значение по умолчанию: false

Название: showWinningBorder
Описание: Признак того, что выигрышные элементы нужно обвести рамкой.
Значение по умолчанию: true

Название: timerRate
Описание: Задержка таймера переключения выигрышных линий между собой (в секундах).
Значение по умолчанию: 1 сек

Название: blinkRate
Описание: Задержка таймера мигания индикаторов линий (в секундах).
Значение по умолчанию: значение атрибута timerRate

Пример
------
<lines count="9" x="0" y="0" width="640" height="366" format="line%d-png" step="2" showTogether="true" showWinningBorder="false" timerRate="1.0" blinkRate="0.4">
- в игре есть 9 линий, находящихся в файлах elements/lines/line1-png - elements/lines/line9-png;
- значение по умолчанию для координат верхнего левого угла (0, 0), размер по умолчанию (640, 366);
- количество линий изменяется по 2: 1, 3, 5, 7 и 9, потом снова 1;
- выигрышные линий будут показывать вместе (рисоваться друг над другом), выигрышные элементы будут обведены рамкой;
- скорость переключения линий - каждую секунду, скорость моргания индикаторов после показа всех выигрышных линий - каждые 0.4 секунды.

Тег: line
Родительский тег: lines
Описание: Предназначен для переопределения некоторых значений для линий, заданных по умолчанию тегом lines. Так же определяет наличие линии с определенным индексом.

Атрибуты:
---------
Название: index
Описание: Индекс линии, которую нужно создать. На каждую линию, которая должна присутствовать на поле, должен присутствовать один тег с индексом данной линии. Индексы начинаются с 0.
          Не может быть больше или равно значения атрибута count тега lines.
Значение по умолчанию: -1

Название: width и height
Описание: Размер данной линии
Значение по умолчанию: значения атрибутов width и height тега lines

Название: x и y
Описание: Координата верхнего левого угла данной линии
Значение по умолчанию: значения атрибутов x и y тега lines

Название: rolls
Описание: Последовательность индексов элементов для каждого барабана, описывающая траекторию линии по полю. Индексы начинаются с 0.
          Например, 1,1,1,1,1 означает, что линия проходит через все вторые элементы барабанов.
Значение по умолчанию: нет

Название: color
Описание: Цвет линии в 16-ричном виде в формате AARRGGBB
Значение по умолчанию: FFFFFFFF (белый)

Пример
------
<line index="0" rolls="1,1,1,1,1" color="FFFF0000"/>
- линия с индексом 0 имеет значения координаты верхнего левого угла и размеров заданные по умолчанию;
- линия проходит через все вторые элементы барабанов (прямая средняя линия);
- цвет линии - красный.

          Не может быть больше или равно значения атрибута count тега lines.
Значение по умолчанию: -1

Название: width и height
Описание: Размер данного индикатора
Значение по умолчанию: значения атрибутов width и height тега indicators

Название: x и y
Описание: Координата верхнего левого угла данного индикатора
Значение по умолчанию: значения атрибутов x и y тега indicators

Пример
------
<indicator index="4"/>
- индикатор линии с индексом 4 имеет значения координаты верхнего левого угла и размеров заданные по умолчанию.

==========================================================================================

Тег: examples
Родительский тег: lines
Описание: Используется, если используются отдельные ресурсы для представления включенных в данный момент линий (если данная опция выключена, то линии будут отрисовываться путем наложения друг на друга с использованием ресурсов из тега lines).

Атрибуты:
---------
Название: useSpecific
Описание: Признак того, что данная опция используется
Значение по умолчанию: false

Название: format и startIndex
Базовая директория: elements/lines/examples
Описание: Описывает группу файлов, формат имени которых представлен в атрибуте format, который обязательно должен содержать подстановку %d, которая будет заменяться числами.
          Количество файлов определяет атрибут count тега lines. Первое число, используемое при считывании файлов, определяется атрибутом startIndex.
          Индекс линии указывает, что данный ресурс будет использован, если количество выбранных линий соответствует индексу линии + 1, т.е. ресурс 2-jpg будет использован, если в данный момент активировано 3 линии.
startIndex: 0
format: нет

Название: width и height
Описание: Размеры по умолчанию считываемых ресурсов
Значение по умолчанию: (0, 0)

Название: x и y
Описание: Координата верхнего левого угла текстуры по умолчанию для всех ресурсов
Значение по умолчанию: (0, 0)

==========================================================================================

Тег: example
Родительский тег: examples
Описание: Предназначен для переопределения некоторых значений для индикаторов линий, заданных по умолчанию тегом indicators. Так же определяет наличие индикатора для линии с определенным индексом.

Атрибуты:
---------
Название: index
Описание: Индекс линии, для которой нужно использовать отдельный ресурс с активными линиями. Индексы начинаются с 0.
          Не может быть больше или равно значения атрибута count тега lines.
Значение по умолчанию: 0

Название: width и height
Описание: Размер ресурса для данного индекса
Значение по умолчанию: значения атрибутов width и height тега examples

Название: x и y
Описание: Координата верхнего левого угла данного ресурса
Значение по умолчанию: значения атрибутов x и y тега examples

Пример
------
<examples useSpecific="false" x="0" y="0" width="640" height="366" format="%d-png" startIndex="1">
	<example index="0"/>
	<example index="2"/>
	<example index="4"/>
	<example index="6"/>
	<example index="8"/>
</examples>
- используются ресурсы с размерами (640, 366) рисуемые в координате (0, 0) и имеющие названия файлов 1-png, 2-png, ..., 9-png;
- ресурсы нужны только для линий с индексами 0, 2, 4, 6 и 8, т.е. когда 1, 3, 5, 7 или 9 линий активно на поле.

==========================================================================================

Тег: textures
Родительский тег: lines
Описание: Служит для группировки тегов с описанием статических ресурсов, расположенных на поле. Порядок тегов, вложенных в данный тег, определяет порядок добавления ресурсов на поле.

==========================================================================================

Тег: textures
Родительский тег: texture
Описание: Содержит информацию о статическом ресурсе, расположенном на поле

Атрибуты:
---------
Название: file
Описание: Путь к файлу с ресуром
Базовый путь: корень архива
Значение по умолчанию: нет


Название: width и height
Описание: Размеры ресурса
Значение по умолчанию: (0, 0)

Название: x и y
Описание: Координата верхнего левого угла расположения данного ресурса на поле
Значение по умолчанию: (0, 0)

Пример
------
<texture file="lines/separators-png" x="0" y="0" width="640" height="366"/>
- текстура из файла lines/separators-png, расположенная в точке (0, 0) и имеющая размеры (640, 366).
